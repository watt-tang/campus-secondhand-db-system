# 校园二手交易信息管理系统

本项目是数据库工程课程作业，主题为“校园二手交易信息管理系统”。系统使用 Python Tkinter 实现桌面端界面，使用 MySQL 保存用户、商品、订单、收藏、评价等业务数据，用于演示数据库建模、完整性约束、视图查询、事务控制以及多表关联操作。

## 项目简介

校园内学生之间经常进行教材、电子产品、生活用品等二手物品交易。该系统将二手交易流程中的用户信息、商品信息、订单信息和用户行为数据集中管理，提供商品浏览、发布、购买、收藏、评价和订单查看等功能。

项目主要包含：

- 学生用户和管理员用户登录
- 商品发布、查询、筛选、删除
- 单商品购买和待下单列表批量结算
- 我的收藏、取消收藏
- 我的订单、订单明细、订单评价
- 商品评价查看和我的评价管理
- MySQL 表、外键、检查约束、唯一约束和视图

## 技术栈

- Python 3
- Tkinter
- MySQL
- PyMySQL

## 数据库设计

系统数据库名为 `campus_secondhand`，核心表如下：

| 表名 | 作用 |
| --- | --- |
| `User` | 用户基础信息表，保存用户名、密码、联系方式、注册时间和用户类型 |
| `StudentUser` | 学生用户子表，保存学号、宿舍、专业等学生信息 |
| `AdminUser` | 管理员用户子表，保存管理员等级和工号 |
| `Category` | 商品类别表，支持父子类别结构 |
| `Product` | 商品表，保存商品名称、价格、库存、状态、描述和发布时间 |
| `OrderInfo` | 订单主表，保存买家、下单时间、总金额、订单状态和收货地址 |
| `OrderItem` | 订单明细表，保存订单和商品之间的明细关系 |
| `Favorite` | 收藏表，实现学生用户和商品之间的多对多收藏关系 |
| `Review` | 评价表，保存订单评价内容和评分 |

`schema.sql` 中还定义了 `v_available_product` 视图，用于封装在售商品、类别和卖家信息的多表查询。`app.py` 中的购买流程会调用 `update_product_after_order` 存储过程来扣减库存并更新状态，新增商品流程也预留了触发器校验说明；如果在全新数据库中演示购买功能，需要确保对应存储过程已经创建。

## 功能模块

### 1. 登录

程序启动时会初始化数据库，然后进入登录窗口。默认演示账号：

| 角色 | 用户名 | 密码 |
| --- | --- | --- |
| 学生用户 | `alice` | `123456` |
| 学生用户 | `bob` | `123456` |
| 管理员 | `admin` | `admin123` |

### 2. 商品管理

登录后进入商品列表主界面，可以按商品名称模糊查询，并按“全部 / 在售 / 已售”筛选。学生用户可以发布商品，商品信息会写入 `Product` 表。

### 3. 购买与订单

学生用户可以购买非本人发布且处于“在售”状态的商品。系统支持：

- 直接购买选中商品
- 加入待下单列表后统一结算
- 生成 `OrderInfo` 订单主表记录
- 生成 `OrderItem` 订单明细记录
- 使用事务保证订单、明细、库存更新一致

### 4. 收藏与评价

学生用户可以收藏商品，收藏记录保存到 `Favorite` 表。购买完成后可以在“我的订单”中对订单进行评价，评价记录保存到 `Review` 表，并且同一个订单最多只能评价一次。

### 5. 事务删除

删除商品时，程序会先检查该商品是否已经被订单明细引用：

- 如果已经出现在 `OrderItem` 中，说明属于历史交易数据，事务回滚，不允许删除
- 如果没有订单引用，则先删除相关收藏记录，再删除商品记录并提交事务

该功能用于演示事务和外键约束下的数据一致性控制。

## 运行方法

### 1. 安装依赖

```bash
pip install pymysql
```

### 2. 配置 MySQL

确认本机 MySQL 服务已启动，并根据实际情况修改 `app.py` 顶部配置：

```python
DB_HOST = '127.0.0.1'
DB_PORT = 3306
DB_USER = 'root'
DB_PASSWORD = '123456'
DB_NAME = 'campus_secondhand'
```

当前代码默认使用 `root / 123456` 连接本机 MySQL。

### 3. 启动程序

方式一：直接运行 Python 文件。

```bash
python app.py
```

方式二：Windows 下双击或执行：

```bat
run_mysql_app.bat
```

首次启动时，程序会自动创建 `campus_secondhand` 数据库，执行 `schema.sql`，并插入基础演示数据。

6. 尝试删除已被订单引用的商品，观察事务回滚提示
7. 切换“在售”筛选，说明该查询使用 `v_available_product` 视图
